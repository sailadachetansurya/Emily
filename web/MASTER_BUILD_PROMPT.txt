PROJECT VISION:
ECHO (Emotional Chronicle Helping observations) is a web platform that combines mental health & wellness 
support with advanced emotional memory tracking & temporal analysis. Users 
journal about their emotional experiences, and the system discovers patterns, 
triggers, relationships, and recovery trajectories over time—providing 
actionable wellness insights with empathy and safety.

================================================================================
SYSTEM ARCHITECTURE OVERVIEW
================================================================================

Frontend:
  - React 18 + TypeScript (SPA)
  - Zustand state management
  - Emotion/Styled Components (theme-aware)
  - Recharts for temporal visualization
  - TailwindCSS + accessibility focus

Backend:
  - Node.js/Express OR Python FastAPI
  - SQLite/PostgreSQL for structured data
  - Vector embeddings for emotional memory
  - JWT authentication
  - Background job queue (Bull/Celery)

Core Systems:
  - Emily AI Integration (emotion engine, policy system)
  - Emotional Memory Graph (embeddings + temporal relationships)
  - Trigger Pattern Mining
  - Recovery Trajectory Analysis
  - Safe Escalation Logic (crisis detection)
  - Relationship Evolution Analyzer

Data Flow:
  [User Input] → [Emily Emotion Engine] → [Emotional Memory Store]
             → [Pattern Analyzer] → [Timeline Visualizer] → [Insights]

TIER 1: MVP ESSENTIALS
─────────────────────────────────────────────────────────────────────────

1. EMOTIONAL JOURNALING INTERFACE
   
   User Story: "I want to write about my feelings and have the system 
   understand what I'm experiencing."
   
   Components:
   - Rich text editor (markdown support)
   - Mood selector (quick emotional tag)
   - Emotion intensity slider (1-10)
   - Optional structured fields:
     * "What triggered this?" (trigger extraction)
     * "Who was involved?" (relationship tagging)
     * "What themes emerge?" (auto-suggested from history)
   - Auto-save draft recovery
   - Privacy indicator (private/shareable)
   
   Emily Integration:
   - Real-time emotion state extraction
   - Confidence scores on emotion classification
   - Speech-act validation (ensure response safety)
   
   Output: EmotionalEntry record
   ```
   {
     id, user_id, timestamp,
     raw_text, emotion_state (valence, activation, stability),
     themes[], triggers[], relationships[],
     ai_summary, confidence, tags
   }
   ```


2. EMOTIONAL MEMORY TIMELINE
   
   User Story: "I want to see how my emotional state has evolved around 
   specific topics or people over weeks and months."
   
   Visualization:
   - Interactive timeline (month/week/day views)
   - Color-coded by emotion (heat map style)
   - Hover to see entry snippet + emotion state
   - Filter by:
     * Date range
     * Emotion type (anxiety, loneliness, joy, etc.)
     * Trigger/theme
     * Relationship
   - Trend indicators (improving/worsening arrows)
   
   Analytics Backing:
   - Emotional state interpolation between entries
   - Volatility calculation (stability metric)
   - Activation trends (arousal over time)
   - Recovery time metrics
   
   Query Layer:
   ```sql
   SELECT entries WITH:
     - Temporal ordering
     - Emotion state aggregation
     - Trend calculation
     - Volatility scoring
   ```


3. TRIGGER MAPPING DASHBOARD
   
   User Story: "I want to understand what situations consistently 
   trigger my anxiety or sadness, and see if I'm improving."
   
   Components:
   - Trigger heatmap (frequency × impact over time)
   - Top triggers list (ranked by frequency + severity)
   - Trigger-to-emotion correlation matrix
   - Trigger evolution graph (reducing/increasing pattern)
   - Mitigation suggestions (based on policy system)
   
   Trigger Extraction:
   - NLP-based extraction from journal entries
   - User confirmation/refinement
   - Categorization (work, relationships, health, financial, etc.)
   - Confidence scoring
   
   Pattern Mining:
   - Weekly/monthly trigger cycles
   - Environmental triggers (time of day, season)
   - Social triggers (specific people/situations)
   - Compound triggers (multiple converging)
   
   Recovery Insights:
   - Average recovery time by trigger type
   - Most effective coping strategies (from entries)
   - Trigger avoidance vs. resilience tracking


4. RELATIONSHIP EMOTIONAL ANALYZER
   
   User Story: "I want to understand how specific relationships are 
   affecting my emotional wellbeing and how they're evolving."
   
   Components:
   - Relationship timeline (emotional tone over time)
   - Sentiment trajectory (positive/negative/neutral trend)
   - Emotional pattern with person (anxiety spikes? joy? trust?)
   - Conflict/resolution cycles
   - Support quality assessment
   - Emotional volatility when relationship is mentioned
   
   Relationship Extraction:
   - Named entity recognition (people mentions)
   - Context analysis (co-occurrence with emotions)
   - Sentiment polarity per mention
   - Trust/safety scoring
   
   Longitudinal Tracking:
   - Timeline shows emotional evolution of relationship
   - Turning points/pivotal moments highlighted
   - Interaction frequency × emotional impact
   - Improvement/deterioration indicators
   
   Insights Generated:
   - "Your emotional stability increased after interactions with Alice"
   - "Conversations with work colleagues trigger anxiety 80% of the time"
   - "This relationship shows improving trust over 3 months"
   - "Recommended: More frequent positive interactions"


5. RECOVERY PROGRESS DASHBOARD
   
   User Story: "I'm in recovery from depression/anxiety and want to see 
   concrete evidence that I'm improving."
   
   Metrics Tracked:
   - Overall emotional baseline (trending up/down)
   - Volatility reduction (emotional stability improving)
   - Good-day frequency (% of days rated above baseline)
   - Recovery time (how fast you bounce back from lows)
   - Trigger resilience (same trigger, but less impact over time)
   - Relationship quality improvements
   
   Visualizations:
   - Progress ring (overall recovery %: 0-100)
   - Weekly mood distribution (box plot)
   - Recovery time reduction graph
   - Stability improvement chart
   - Milestone celebrations (first trigger-free week, etc.)
   - Goal tracking (user-set wellness goals)
   
   Wellness Goals:
   - "Reduce anxiety around performance reviews"
   - "Rebuild trust with sister"
   - "Achieve 3 consecutive good days"
   - "Keep emotional volatility under 0.5"
   - Progress tracking with timeline
   - Smart recommendations to achieve goals

Database Schema:
  
  users
    ├── id (UUID)
    ├── email (unique)
    ├── password_hash
    ├── name
    ├── therapy_mode (boolean: are they in active therapy?)
    ├── created_at, updated_at
    └── settings (JSON: privacy, notification preferences)
  
  journal_entries
    ├── id (UUID)
    ├── user_id (FK)
    ├── timestamp
    ├── raw_text
    ├── emotion_state (JSON: valence, activation, stability)
    ├── themes (JSON array: extracted themes)
    ├── triggers (JSON array: extracted triggers)
    ├── relationships_mentioned (JSON array: people)
    ├── ai_summary (auto-generated)
    ├── ai_confidence (0-1)
    ├── is_private (boolean)
    ├── created_at, updated_at
    └── deleted_at (soft delete)
  
  emotional_memories
    ├── id (UUID)
    ├── user_id (FK)
    ├── journal_entry_id (FK)
    ├── summary (extracted emotional core)
    ├── embedding (vector: 32-dim)
    ├── themes (array)
    ├── triggers (array)
    ├── recency_weight (float)
    ├── created_at
    └── expires_at (optional: memory fading)
  
  triggers
    ├── id (UUID)
    ├── user_id (FK)
    ├── trigger_text (e.g., "work meetings")
    ├── category (work, relationship, health, etc.)
    ├── confidence (0-1: extraction confidence)
    ├── frequency_count
    ├── avg_emotional_impact (0-1)
    ├── recovery_time_avg (minutes)
    ├── is_confirmed (user validated)
    ├── created_at, updated_at
    └── is_active (user tracking?)
  
  relationships
    ├── id (UUID)
    ├── user_id (FK)
    ├── person_name
    ├── relationship_type (friend, family, partner, coworker, etc.)
    ├── mentioned_count
    ├── sentiment_recent (positive/negative/neutral)
    ├── trust_score (0-1: derived from entries)
    ├── emotional_impact_score (how much they affect mood)
    ├── created_at, updated_at
    └── last_mentioned
  
  recovery_goals
    ├── id (UUID)
    ├── user_id (FK)
    ├── goal_text
    ├── goal_type (reduce_trigger, rebuild_relationship, improve_stability, etc.)
    ├── target_date
    ├── progress_percent
    ├── milestones (JSON array with dates)
    ├── created_at, updated_at
    └── completed_at
  
  insights
    ├── id (UUID)
    ├── user_id (FK)
    ├── insight_type (pattern, trigger_correlation, relationship_evolution, recovery_progress)
    ├── title
    ├── description
    ├── confidence (0-1)
    ├── supporting_data (JSON)
    ├── actionable_recommendation
    ├── created_at
    └── dismissed_at (user dismissed insight)
  
  escalation_events
    ├── id (UUID)
    ├── user_id (FK)
    ├── event_type (low_mood_sustained, isolation_increase, crisis_indicators)
    ├── severity (low, medium, high, critical)
    ├── triggered_at
    ├── acknowledged_at
    ├── action_taken
    ├── resource_provided
    └── is_resolved

================================================================================
KEY DESIGN PRINCIPLES
================================================================================

1. EMPATHY FIRST
   - Never judgmental language
   - Celebrate small wins
   - Offer compassion, not criticism
   - Agency always with user

2. TRANSPARENCY
   - Show why Emily flagged something
   - Explain insights (not black-box AI)
   - User can refine/correct AI interpretations
   - Clear data usage policy

3. SAFETY
   - Crisis detection with escalation
   - Resource provision (never just warning)
   - Privacy by default
   - Therapist-informed design

4. ACCESSIBILITY
   - WCAG 2.1 AA minimum
   - Keyboard navigation
   - Screen reader support
   - High contrast options
   - Simple language option

5. PRIVACY
   - User owns their data
   - Minimal tracking
   - Consent-based sharing
   - Right to delete

6. EVIDENCE-BASED
   - Grounded in psychology research
   - Patterns based on actual data
   - Interventions from CBT/DBT/ACT
   - Validated with therapists
